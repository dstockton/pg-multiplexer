# PgMux v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename pg-multiplexer to PgMux, replace hand-rolled auth with postgres-protocol, migrate to prometheus-client, ship multi-arch distroless Docker images to GHCR, update all deps, and make the project adoptable.

**Architecture:** The binary and crate rename from `pg-multiplexer` to `pgmux`. Backend auth delegates to `postgres-protocol` while keeping the custom wire-protocol proxy loop. Metrics migrate from `prometheus` to `prometheus-client`. Docker images use `gcr.io/distroless/cc-debian12:nonroot` and publish to GHCR via buildx.

**Tech Stack:** Rust, tokio, postgres-protocol, prometheus-client, axum, Docker buildx, GHCR

**Spec:** `docs/superpowers/specs/2026-03-24-pgmux-v2-design.md`

## Task 2: Backend Auth — postgres-protocol

Replace hand-rolled SCRAM/MD5/PBKDF2 in `backend.rs` with `postgres-protocol` crate.

**Files:**
- Modify: `src/protocol/backend.rs`

**Context for implementer:** Read `src/protocol/backend.rs` fully before starting. The function `connect_backend()` opens a TCP connection, sends a startup message, then handles auth responses. The auth handling (match on auth_type 3/5/10) and all helper functions (`compute_md5_password`, `handle_scram_auth`, `pbkdf2_sha256`) get replaced. Everything else (`reset_connection`, `health_check`, `parse_error_fields`) stays unchanged.

- [ ] **Step 1: Read current backend.rs to understand the auth flow**

Read `src/protocol/backend.rs` in full. Note the auth_type match at lines 45-83 and the helper functions below.

- [ ] **Step 2: Replace auth handling in connect_backend**

In `connect_backend()`, replace the `match auth_type` block:

For auth_type 3 (CleartextPassword): Use `postgres_protocol::message::frontend::password_message(password)` to build the password message bytes, write them to the stream.

For auth_type 5 (MD5): Extract the 4-byte salt from `msg.payload[4..8]`. Use `postgres_protocol::authentication::md5_hash(user.as_bytes(), password.as_bytes(), salt)` to compute the hash. Send via `postgres_protocol::message::frontend::password_message(&hash)`.

For auth_type 10 (SASL/SCRAM-SHA-256): Replace the entire `handle_scram_auth` function body with `postgres_protocol::authentication::sasl`:
1. Parse mechanism list from payload, confirm SCRAM-SHA-256 is available
2. Create `ScramSha256::new(password.as_bytes(), ChannelBinding::unsupported())`
3. Call `scram.message()` to get client-first bytes
4. Use `postgres_protocol::message::frontend::sasl_initial_response("SCRAM-SHA-256", client_first, &mut buf)` to frame the SASLInitialResponse — this writes directly to a `BytesMut`
5. Read SASLContinue (auth_type 11), call `scram.update(server_first_bytes)`
6. Call `scram.message()` to get client-final bytes
7. Use `postgres_protocol::message::frontend::sasl_response(client_final, &mut buf)` to frame the SASLResponse
8. Read SASLFinal (auth_type 12), call `scram.finish(server_final_bytes)`

- [ ] **Step 3: Remove dead code**

Delete these functions entirely:
- `compute_md5_password`
- `handle_scram_auth`
- `pbkdf2_sha256`

Remove unused imports: `hmac`, `sha2`, `md5`, `base64`, `rand` (check if `rand` is still used — it is in `frontend.rs` for `BackendKeyData`, so keep it in the crate but remove from this file's imports).

## Task 5: Dockerfile — Distroless Multi-arch

**Files:**
- Modify: `Dockerfile`

- [ ] **Step 1: Read current Dockerfile**

Read `Dockerfile` to understand current structure.

- [ ] **Step 2: Rewrite Dockerfile**

```dockerfile
# =============================================================================
# pgmux Docker build — multi-arch via buildx
# =============================================================================

# Build stage: use host platform for speed, cross-compile for target
FROM --platform=$BUILDPLATFORM rust:1-bookworm AS builder

ARG TARGETARCH

# Install cross-compilation tools for arm64
RUN if [ "$TARGETARCH" = "arm64" ]; then \
      apt-get update && apt-get install -y gcc-aarch64-linux-gnu && \
      rm -rf /var/lib/apt/lists/*; \
    fi

WORKDIR /app

# Map Docker arch to Rust target triple
RUN case "$TARGETARCH" in \
      amd64) echo "x86_64-unknown-linux-gnu" > /tmp/rust-target ;; \
      arm64) echo "aarch64-unknown-linux-gnu" > /tmp/rust-target ;; \
      *) echo "unsupported: $TARGETARCH" && exit 1 ;; \
    esac && \
    rustup target add $(cat /tmp/rust-target)

# Configure linker for cross-compilation
RUN if [ "$TARGETARCH" = "arm64" ]; then \
      mkdir -p .cargo && \
      echo '[target.aarch64-unknown-linux-gnu]' > .cargo/config.toml && \
      echo 'linker = "aarch64-linux-gnu-gcc"' >> .cargo/config.toml; \
    fi

# Cache dependency build
COPY Cargo.toml Cargo.lock ./
RUN mkdir src && echo 'fn main() {}' > src/main.rs && \
    echo 'pub mod admin; pub mod auth; pub mod config; pub mod monitor; pub mod pool; pub mod protocol; pub mod tls;' > src/lib.rs && \
    mkdir -p src/admin src/auth src/protocol src/monitor src/pool src/tls && \
    touch src/admin/mod.rs src/admin/metrics.rs src/admin/server.rs src/auth/mod.rs src/config.rs src/monitor/mod.rs src/pool/mod.rs src/protocol/mod.rs src/protocol/backend.rs src/protocol/frontend.rs src/protocol/messages.rs src/tls/mod.rs
RUN cargo build --release --target $(cat /tmp/rust-target) 2>/dev/null || true
RUN rm -rf src

# Build the actual application
COPY src/ src/
RUN touch src/main.rs && cargo build --release --target $(cat /tmp/rust-target)

# Move binary to predictable location
RUN cp target/$(cat /tmp/rust-target)/release/pgmux /usr/local/bin/pgmux

# Runtime stage: distroless with glibc
FROM gcr.io/distroless/cc-debian12:nonroot

COPY --from=builder /usr/local/bin/pgmux /usr/local/bin/pgmux
COPY config.toml /etc/pgmux/config.toml

EXPOSE 5433 9090

ENTRYPOINT ["pgmux"]
CMD ["--config", "/etc/pgmux/config.toml"]
```

```yaml
    name: Docker Build & Push
    runs-on: ubuntu-latest
    needs: [check, unit-tests]
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Docker metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/dstockton/pgmux
          tags: |
            type=sha
            type=ref,event=branch
            type=semver,pattern={{version}}
            type=raw,value=latest,enable=${{ github.ref == format('refs/heads/{0}', 'main') }}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          platforms: linux/amd64,linux/arm64
          push: ${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```
